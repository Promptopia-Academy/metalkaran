# راهنمای پنل مدیریت

## دسترسی

برای دسترسی به پنل مدیریت، به آدرس زیر بروید:

```
http://localhost:3000/admin
```

## تنظیمات اولیه

### 1. تنظیم API Key

1. به صفحه **تنظیمات** بروید (`/admin/settings`)
2. API Key خود را از فایل `.env.local` سرور کپی کنید
3. API Key را در فیلد مربوطه وارد کنید
4. روی دکمه **ذخیره** کلیک کنید

**نکته:** API Key در مرورگر شما ذخیره می‌شود و برای احراز هویت با API استفاده می‌شود.

## بخش‌های مختلف

### داشبورد (`/admin`)
- نمایش آمار کلی
- تعداد مقالات
- تعداد المنت‌ها
- تعداد پیام‌های تماس
- وضعیت سیستم

### مقالات (`/admin/articles`)
- مشاهده لیست مقالات
- جستجو در مقالات
- ایجاد مقاله جدید
- ویرایش مقاله
- حذف مقاله

### المنت‌ها (`/admin/elements`)
- مشاهده لیست المنت‌ها
- جستجو در المنت‌ها
- ایجاد المنت جدید
- ویرایش المنت
- حذف المنت

### پیام‌های تماس (`/admin/contacts`)
- مشاهده تمام پیام‌های دریافتی
- اطلاعات تماس کاربران
- تاریخ و زمان دریافت پیام

### تنظیمات (`/admin/settings`)
- تنظیم API Key
- راهنمای استفاده
- نکات امنیتی

## ویژگی‌ها

### ✅ فونت Vazir
- تمام صفحات پنل از فونت Vazir استفاده می‌کنند
- پشتیبانی کامل از RTL

### ✅ تم موجود
- استفاده از تم و رنگ‌های موجود در پروژه
- Sidebar با رنگ Primary
- Cards و Buttons با استایل موجود

### ✅ مدیریت کامل
- CRUD کامل برای Articles و Elements
- مشاهده و مدیریت Contacts
- Pagination برای لیست‌ها
- جستجو و فیلتر

### ✅ امنیت
- احراز هویت با API Key
- ذخیره‌سازی امن در localStorage
- بررسی API Key قبل از درخواست‌ها

## نحوه استفاده

### ایجاد مقاله جدید

1. به بخش **مقالات** بروید
2. روی دکمه **مقاله جدید** کلیک کنید
3. فرم را پر کنید (فیلدهای * الزامی هستند)
4. روی **ذخیره مقاله** کلیک کنید

### ویرایش مقاله

1. در لیست مقالات، روی دکمه **ویرایش** کلیک کنید
2. اطلاعات را ویرایش کنید
3. روی **ذخیره** کلیک کنید

### حذف مقاله

1. در لیست مقالات، روی دکمه **حذف** کلیک کنید
2. در پیام تأیید، روی **تأیید** کلیک کنید

## نکات مهم

1. **API Key ضروری است:** بدون API Key نمی‌توانید مقالات یا المنت‌ها را ایجاد/ویرایش/حذف کنید
2. **ذخیره‌سازی محلی:** API Key در localStorage مرورگر ذخیره می‌شود
3. **امنیت:** API Key را با کسی به اشتراک نگذارید
4. **Backup خودکار:** قبل از هر عملیات write، به صورت خودکار backup ایجاد می‌شود
